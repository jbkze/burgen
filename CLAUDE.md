## Overview

**Burgen in Deutschland** — a fully static site (castle browser + trip planner) served via **GitHub Pages** at **burgen.jbkze.de**. There is no backend: all dynamic features run as JavaScript in the visitor's browser against public CORS-enabled APIs.

See PRODUCT.md for product goals and design principles.

## Architecture

- `site/` — the deployed site, exactly as served by GitHub Pages
  - `site/index.html` — the entire frontend (single file: markup, CSS, JS). Loads `data/castles_enriched.json` statically; filtering/browsing is client-side.
  - `site/data/` — castle data (generated by the pipeline below, committed)
- Trip planner calls external APIs directly from the browser:
  - `https://v6.db.transport.rest` (HAFAS/DB) — station lookup and train journeys
  - `https://router.project-osrm.org` — driving time/distance matrix (also used to pre-filter train candidates before the slower HAFAS calls)
- `scraper.py` — scrapes castle data from ebidat.de into `site/data/castles.json`
- `enrich.py` — adds Komoot hike counts and nearest train stations, writes `site/data/castles_enriched.json`

## Deployment

Push to `main` → GitHub Actions (`.github/workflows/deploy.yml`) uploads `site/` verbatim to GitHub Pages. Custom domain `burgen.jbkze.de` is configured in the repo's Pages settings; DNS is a CNAME in Cloudflare pointing to `jbkze.github.io`.

## Local development

```bash
python -m http.server -d site
```
